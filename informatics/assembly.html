🖥️ Langage Assembleur (ASM)

Le langage assembleur est un langage de bas niveau, spécifique à chaque architecture de processeur (x86, ARM, MIPS). Il offre un contrôle absolu sur le matériel, crucial pour l'embarqué, la rétro-ingénierie et l'optimisation extrême.

01. Pré-requis Système

  • Architecture : Compréhension du jeu d'instructions cible (x86 vs ARM).
  • Assembleur : Outil de traduction (ex: NASM pour Windows/Linux, as pour GNU).
  • Débogueur : Outil d'analyse d'exécution (ex: GDB).

02. Installation (NASM)

# Linux (Debian/Ubuntu) sudo apt update && sudo apt install nasm # Windows Download: https://www.nasm.us/

03. Structure & Syntaxe

Un programme ASM s'articule généralement autour de trois segments :

  • Section .data : Variables initialisées et constantes.
  • Section .bss : Variables non-initialisées (mémoire réservée).
  • Section .text : Code exécutable (instructions).

Exemple : Hello World (Linux x86 32-bit)

section .data hello db 'Hello, World!',0 ; Chaine de caractères section .text global _start _start: ; --- Affichage via sys_write --- mov eax, 4 ; Appel système 4 (write) mov ebx, 1 ; File descriptor 1 (stdout) mov ecx, hello ; Pointeur vers le texte mov edx, 13 ; Longueur int 0x80 ; Interruption Kernel ; --- Sortie via sys_exit --- mov eax, 1 ; Appel système 1 (exit) xor ebx, ebx ; Code de retour 0 int 0x80 ; Interruption Kernel

04. Compilation & Liaison

Step 1: Assemblage en fichier objet (.o)

nasm -f elf32 -o hello.o hello.asm

Step 2: Liaison (Linker)

ld -m elf_i386 -s -o hello hello.o

Step 3: Exécution

./hello

05. Instructions Fondamentales

  • mov [dest], [src] : Transfert de données.
  • add / sub : Opérations arithmétiques.
  • cmp [v1], [v2] : Comparaison de valeurs.
  • je / jne / jmp : Sauts conditionnels ou directs (branchements).
  • int / syscall : Interaction avec le noyau (API système).

06. Débogage (GDB)

Essentiel pour inspecter les registres et la pile (stack) :

gdb ./hello (gdb) break _start (gdb) run (gdb) info registers (gdb) stepi
